\section*{Project Summary}

This project investigates an application-derived approach to dynamic voltage and frequency scaling (DVFS) for GPU-based high-performance computing workloads. In contrast to traditional system-level DVFS mechanisms, which rely on indirect hardware signals such as power consumption, temperature, or utilization, this work proposes a runtime control framework that derives optimization signals directly from the application itself. The core idea is to expose application-level execution semantics—such as training loop structure, kernel execution characteristics, and runtime performance indicators—through a Python decorator-based instrumentation layer. This layer enables fine-grained monitoring and control of GPU power states during execution of PyTorch-based deep learning workloads. Based on these signals, a heuristic control policy dynamically adjusts GPU frequency and power settings using available runtime interfaces (e.g., NVML and system-level power management APIs). The system is evaluated in terms of energy consumption, performance overhead, and training efficiency compared to conventional GPU DVFS governors. The goal of this work is to assess whether application-aware runtime information can improve energy-performance trade-offs in GPU computing and to explore the feasibility of integrating such control mechanisms directly into scientific software stacks.

\section{Introduction}

Since the explosion of AI, many hyperscalers, as well as research institutions, heavily invest in GPU based research and model training. Hardware manufacturers thus lean very heavily into broad vector and matrix centric calculation accelerators, most notably Graphics-Processing-Units. Given that the market demand mostly focuses on maximum performance per dollar, many other key factors are not considered, such as energy efficiency. However, not all calculations need to be run at the best performance; they can also be run at the most optimal balance between performance and energy cost. The ever growing cost and demand for energy propose the necessity to also optimise for said factors. The goal of this work is to create a Python based framework that implements different energy optimisation algorithms and uses them to adjust the GPUs Voltage and frequency settings to create a proper balance. The idea is to optimise for a key that, as opposed to traditional methods, gathers its information from the application itself. This means that optimisation keys can be determined by the developer of the scientific application and thus are easily changeable during runtime as well as the development cycle.

\section{Theoretical Framework}

The underlying problem is a classical mathematical optimisation problem. Only a few parameters are needed to decide on the next iteration steps to change the GPU parameters since the key comes from the application itself. A simple example would be:
\begin{itemize}
    \item $f_t \in \mathbb{R}$: GPU frequency (MHz) at time step $t$
    \item $k \in \{\text{key}, \text{runtime}\}$: optimisation metric
    \item $R(f_t, k)$: performance evaluation function (higher is better)
    \item $t \in \mathbb{N}$: iteration index of the optimisation process
\end{itemize}

\textbf{Objective:}

\[
f^{*} = \arg\max_f \; R(f, k)
\]

\textbf{Update rule (discrete hill-climbing):}

\[
f_{t+1} =
\begin{cases}
f_t + 50 & \text{if } R(f_t, k) > R(f_{t-1}, k) \\
f_t - 50 & \text{otherwise}
\end{cases}
\]
Here, a simple step function is used, which brings some problems by itself, such as a high amount of jitter around the local minimum.
\section{Traditional Approach and Key Innovations}
Traditionally, DVFS systems rely on the input variables of the current power draw, temperature, and resource utilisation. This introduces the problem that these variables sometimes do not directly influence the actual problem as strongly as they scale up the power draw. This highlights the need for a better heuristic that is directly linked to the actual application performance. Since this is only possible by explicitly stating these variables during runtime, it needs to be incorporated by the developer. The key innovation of this work is a novel optimisation algorithm that uses a combination of traditional optimisers such as gradient descent and tries to measure its performance in comparison to other optimisation functions.
